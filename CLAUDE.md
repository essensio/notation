# Правила письма — репозиторий нотации

Нормативные тексты репозитория (`grammar.ebnf`, `spec/`, `examples/`, `README.md`) подчиняются правилам ниже. Нарушение — грубая ошибка.

## 1. Без JSON

Не упоминать **JSON** ни в какой форме («JSON-документ», «JSON-объект», «JSON-массив», «надмножество JSON», «совместимость с JSON», «JSON-чисел» и т. п.).

Нотация — **абстрактная система типов**. Связь с конкретными форматами ввода (JSON и прочие) — забота потребителей (например, схематора), а не самой нотации. Явления вроде разнородного массива, нечитаемого/произвольного ключа, пустого объекта описывай **абстрактно**, по существу системы типов, без отсылки к формату.

## 2. Без конструкта «раньше … теперь»

Не описывать систему через изменение во времени: «раньше X — теперь Y», «прежде… теперь…», «больше нет», «стало». Нормативный текст вне времени — описывай **конечное состояние в настоящем**, не вводи понятие через контраст с прежним поведением.
